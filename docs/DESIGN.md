# DESIGN.md — визуальная спецификация KASTOL

> Это **обязательная** спецификация для любого UI-кода. Все цвета, размеры,
> отступы, типографика — отсюда. Не из головы. Если в спеке нет — спрашиваем,
> не выдумываем. Цель — выглядеть как продукт уровня Linear / Vercel / Raycast,
> а не как обычный читерский оверлей.

## §1. Дизайн-принципы

1. **Минимализм без бедности.** Меньше элементов — но каждый качественный.
   Чёткие границы, правильные тени, чистые иконки.
2. **Контраст вместо украшений.** Чёрный текст на белом фоне. Один акцентный
   цвет. Никаких градиентов, glow, свечений, неона.
3. **Воздух важнее плотности.** Лучше пустое место, чем напихать элементы.
   Padding щедрый.
4. **Иконки — чёрные, тонкие, одного семейства.** Никаких эмодзи в продакт-UI.
5. **Анимации сдержанные.** 150-200ms ease-out на hover/toggle. Без bounce.
6. **Шрифт один.** Inter (или Segoe UI Variable как fallback).

## §2. Цветовая палитра

Светлая тема — **базовая** (по запросу пользователя):

| Назначение | Имя в коде | HEX | RGB | Пример использования |
|---|---|---|---|---|
| Фон окна | `BG_PRIMARY` | `#FFFFFF` | 255 255 255 | Body окна |
| Фон сайдбара | `BG_SECONDARY` | `#F7F7F8` | 247 247 248 | Левая панель |
| Hover поверхность | `BG_HOVER` | `#EFEFF1` | 239 239 241 | Hover на кнопках/айтемах |
| Active поверхность | `BG_ACTIVE` | `#E5E5E8` | 229 229 232 | Выбранная вкладка фон |
| Border subtle | `BORDER` | `#E5E5E8` | 229 229 232 | Разделители 1px |
| Текст основной | `TEXT_PRIMARY` | `#0A0A0A` | 10 10 10 | Заголовки, значения |
| Текст вторичный | `TEXT_SECONDARY` | `#6B6B6B` | 107 107 107 | Подписи, описания |
| Текст disabled | `TEXT_DISABLED` | `#A1A1A1` | 161 161 161 | Неактивные элементы |
| **Акцент** | `ACCENT` | `#0A0A0A` | 10 10 10 | **Чёрный — основной акцент по умолчанию** |

> Важно: «акцентный цвет» в KASTOL по умолчанию **чёрный**. Это и есть
> минимализм — кнопки primary тёмные, фон белый. Контраст максимальный.
> Пользователь сможет выбрать другой акцент в Visual вкладке (см. §6).

### Альтернативные акценты (выбор пользователя)

| Имя | HEX | Когда применять |
|---|---|---|
| Black (default) | `#0A0A0A` | Дефолт, максимальный контраст |
| Indigo | `#4F46E5` | Профессионально, нейтрально |
| Emerald | `#059669` | Свежо |
| Rose | `#E11D48` | Акцентно, для warning |
| Amber | `#D97706` | Тёплый |

### Состояния акцента

```
ACCENT          = выбранный цвет
ACCENT_HOVER    = ACCENT, но lighten 5%
ACCENT_PRESSED  = ACCENT, но darken 5%
ACCENT_TEXT     = #FFFFFF (текст на акцентном фоне)
```

## §3. Размеры и отступы

База: **8px grid**. Все отступы кратны 4 или 8.

| Token | Значение | Где |
|---|---|---|
| `RADIUS_SM` | 6px | Кнопки, инпуты, чипы |
| `RADIUS_MD` | 10px | Карточки, сабпанели |
| `RADIUS_LG` | 12px | Главное окно |
| `PAD_XS` | 4px | Внутри иконки |
| `PAD_SM` | 8px | Минимальный внутренний |
| `PAD_MD` | 16px | Стандартный (контент) |
| `PAD_LG` | 24px | Между секциями |
| `PAD_XL` | 32px | Внешние отступы окна |
| `BORDER_W` | 1px | Все разделители |

### Размер главного окна

- Ширина: **720px**
- Высота: **480px**
- Минимальный размер: тот же (не ресайзится по умолчанию).
- Позиция при первом открытии: центр экрана.
- Сохраняется последняя позиция в конфиге.

### Сайдбар

- Ширина: **200px**.
- Padding: 16px 12px.
- Между айтемами: 4px вертикальный gap.

### Контентная область

- Ширина: оставшаяся (520px).
- Padding: 32px.

### Кнопки

| Тип | Высота | Padding по горизонтали |
|---|---|---|
| Default | 36px | 16px |
| Compact (в формах) | 28px | 12px |
| Icon-only | 32×32px | 0 |

## §4. Типографика

**Шрифт:** Inter Variable (либо Inter Regular/Medium/SemiBold).
Встраиваем в DLL как ресурс через `.rc` файл и `LoadResource`/`AddFontFromMemoryTTF`.
Лицензия Inter — OFL (можно встраивать).

| Стиль | Размер | Weight | Letter spacing | Где |
|---|---|---|---|---|
| `H1` (заголовок окна) | 18px | 600 (SemiBold) | -0.01em | «KASTOL» в шапке |
| `H2` (заголовок секции) | 15px | 600 | -0.005em | Заголовки в табах |
| `Body` | 13px | 400 (Regular) | 0 | Основной текст |
| `Body-Medium` | 13px | 500 (Medium) | 0 | Лейблы инпутов |
| `Caption` | 11px | 400 | 0.01em | Подписи, версии, watermark |
| `Mono` | 12px | 400 | 0 | Версия, hex-цвета |

> ImGui по умолчанию рендерит на 13-14px. Все размеры должны масштабироваться
> через `io.FontGlobalScale` от размера окна CS2 — например, 1.0 для 1080p,
> 1.25 для 1440p, 1.5 для 4K. Это считается через DPI.

## §5. Иконки

**Семейство:** [Lucide](https://lucide.dev) (рекомендуется) или Tabler Icons.
Подключение: через **IconFontCppHeaders** (`IconsLucide.h`).

Правила:
- Размер в UI: **16px** в сайдбаре, **14px** инлайн с текстом.
- Stroke width: 1.5px (по умолчанию у Lucide — то что нужно).
- Цвет: `TEXT_PRIMARY` (чёрный) или `TEXT_SECONDARY` (серый) для disabled.
- Никогда не цветные. Никаких filled-вариантов.
- Расстояние между иконкой и текстом: 12px.

Конкретные иконки для вкладок сайдбара:
- Home → `home`
- Visual → `palette`
- Performance → `activity` или `gauge`
- About → `info`

## §6. Компоненты

### 6.1. Главное окно

```
┌──────────────────────────────────────────────────┐
│  ⬛ KASTOL                              v1.0.0  │  ← заголовок (H1 + Caption)
├──────────────────────────────────────────────────┤
│            │                                     │
│  🏠 Home   │   Заголовок секции (H2)             │
│  🎨 Visual │                                     │
│  📊 Perf   │   Контент. PAD_LG между блоками.   │
│  ℹ️  About │                                     │
│            │                                     │
│  ┌──────┐  │                                     │
│  │ ↻    │  │                                     │
│  │ Reset│  │                                     │
│  └──────┘  │                                     │
└──────────────────────────────────────────────────┘
```

- Скруглённые углы окна `RADIUS_LG`.
- Тень: НЕ рисуем (ImGui сам не умеет красиво — выглядит дёшево).
- Заголовочная полоса 48px, по центру вертикально.
- Закрывающего крестика **нет** — закрытие только через INSERT (это оверлей,
  не обычное окно).

### 6.2. Sidebar item

```
States:
  default:  text=PRIMARY, bg=transparent
  hover:    text=PRIMARY, bg=BG_HOVER
  active:   text=ACCENT,  bg=BG_ACTIVE, leading 2px ACCENT bar слева

Layout:
  [icon 16px] [12px gap] [text 13/Medium]
  height 36px, padding 0 12px, radius 6px
```

### 6.3. Кнопки

**Primary** (например «Save»):
```
bg=ACCENT, text=ACCENT_TEXT, radius=RADIUS_SM
hover: bg=ACCENT_HOVER
pressed: bg=ACCENT_PRESSED
```

**Secondary** (например «Cancel»):
```
bg=BG_PRIMARY, text=TEXT_PRIMARY, border=1px BORDER, radius=RADIUS_SM
hover: bg=BG_HOVER
```

**Ghost** (для toolbar):
```
bg=transparent, text=TEXT_PRIMARY, radius=RADIUS_SM
hover: bg=BG_HOVER
```

### 6.4. Toggle (свитч)

- Стиль iOS-like, 36×20px.
- Off: `bg=BG_ACTIVE, knob=BG_PRIMARY`.
- On: `bg=ACCENT, knob=ACCENT_TEXT`.
- Анимация перемещения 150ms ease-out.

### 6.5. Slider

- Track: 4px высотой, `BG_ACTIVE`.
- Filled track: `ACCENT`.
- Knob: 16×16, white круг, `border 1px BORDER`.
- Подпись текущего значения справа от слайдера, `Caption`.

### 6.6. Color picker (для выбора акцента)

5 кружков 24×24, расположены в ряд, gap 12px:
```
( ⚫ ) ( 🟣 ) ( 🟢 ) ( 🔴 ) ( 🟠 )
```
Выбранный — кольцо `BORDER` 2px вокруг.

### 6.7. Tooltip

- bg=`#0A0A0A` (тёмный) — единственное исключение из светлой темы.
- text=`#FFFFFF`, размер `Caption`, padding 8px 10px, radius `RADIUS_SM`.
- Появляется через 400ms hover, fade-in 100ms.

## §7. FPS-счётчик (watermark)

```
12px Caption, ACCENT цвет, opacity 0.7
"KASTOL · 144 fps"
позиция: один из углов экрана (config), 16px от края
```

Без фоновой плашки. Просто текст.

## §8. Анимации

| Действие | Длительность | Easing |
|---|---|---|
| Hover (цвет) | 150ms | ease-out |
| Toggle меню (fade) | 200ms | ease-out |
| Tab switch (cross-fade) | 150ms | ease-in-out |
| Toggle switch (knob) | 150ms | ease-out |
| Tooltip fade-in | 100ms | ease-out |

ImGui не имеет встроенных анимаций — реализуем через интерполяцию по `deltaTime`
в `Overlay::Render()`. Простой подход: `value = lerp(value, target, dt * speed)`.

## §9. Чего НЕ делаем

- ❌ Никаких градиентов.
- ❌ Никакого glow / свечения / blur (если только это не системный
  back-drop blur Windows 11 — но это сложно через ImGui).
- ❌ Никаких эмодзи в финальном UI (✓ ⚙️ ❌). Только Lucide.
- ❌ Никаких анимированных фонов / partikle-эффектов.
- ❌ Никакого скевоморфизма (3D кнопок, теней-выпуклостей).
- ❌ Никаких иконок разного семейства / разной толщины.
- ❌ Никаких ярких заливок (всё кричаще-цветное).
- ❌ **Никаких функций, влияющих на геймплей.** Это оверлей-визуал.

## §10. Reference (вдохновение)

Цельтесь визуально на:
- **Linear** (linear.app) — карточки, типографика, отступы.
- **Vercel Dashboard** (vercel.com) — кнопки, формы, contrast.
- **Raycast** (raycast.com) — sidebar, command palette feel.
- **Figma** — toolbar, color picker, плотность.

Всё это — белый/светлый фон, чёрные иконки, акцент в виде одного цвета,
ноль украшательств. Это и есть наш референс.

## §11. Чек-лист «выглядит правильно»

Перед мерджем PR с UI-изменениями убедиться:

- [ ] Все цвета — из палитры §2, никаких произвольных hex.
- [ ] Все размеры — из §3, кратны 4.
- [ ] Шрифт — Inter (не дефолтный ImGui Proggy).
- [ ] Иконки — все из одного семейства (Lucide).
- [ ] Hover/active работают, цвета по спеке.
- [ ] При наложении на CS2 (любой скриншот игры) UI остаётся читаемым.
- [ ] Скриншот приложен к PR.
